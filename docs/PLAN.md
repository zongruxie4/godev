# PLAN: Usar `tinywasm/json` + `ormc` para JSON en `tinywasm/app`

## Problema

`tinywasm/app/daemon.go` construye la respuesta JSON-RPC de `tinywasm/state` con
`tinywasm/fmt.Sprintf` y el verbo `%q`:

```go
// daemon.go ~línea 152
resp := fmt.Sprintf(`{"jsonrpc":"2.0","id":%s,"result":%q}`, id, string(stateJSON))
```

**Defecto 1 — `%q` no escapa caracteres especiales:**
`stateJSON` es un array JSON que contiene comillas (`"`). El `%q` producía JSON inválido
→ `tinywasm/json.Decode` falla silenciosamente → `reconstructRemoteHandlers` nunca se llama
→ **footer TUI permanece vacío**.
_(Fix de `%q` ya aplicado en `tinywasm/fmt/docs/PLAN_FIX_Q_VERB_ESCAPE.md`)_

**Defecto 2 — violación de política del ecosistema:**
Todo JSON en tinywasm debe pasar por `tinywasm/json`. La interpolación manual de strings
para construir JSON bypasea la cobertura de la librería.

> El fix del lado cliente (`tinywasm/mcp`) está documentado en
> `tinywasm/mcp/docs/PLAN.md` — ese plan es independiente y debe ejecutarse por separado.

---

## Estrategia: `ormc` genera el código, `tinywasm/json` serializa

El campo `fmt.RawJSON` (alias de `string`) indica a `ormc` que genere `FieldRaw` en
el `Schema()`. `tinywasm/json` emite el valor verbatim (sin re-serializar), exactamente
como `encoding/json.RawMessage`. Así `stateJSON` llega al cliente como array JSON directo,
sin doble encoding.

## Archivos afectados

| Archivo | Acción |
|---|---|
| `app/model.go` | **Crear** — struct `stateResponse` con `// ormc:formonly` |
| `app/model_orm.go` | **Generar** con `ormc` |
| `app/daemon.go` | Reemplazar `fmt.Sprintf %q` con `twjson.Encode(&stateResponse{...})` |

---

## Paso 1 — Crear `app/model.go`

`ormc` escanea `model.go` / `models.go` y genera `model_orm.go`.
La directiva `// ormc:formonly` indica que es solo para JSON/formularios (sin tabla DB).
`ormc` detecta `fmt.RawJSON` automáticamente y genera `FieldRaw` en el `Schema()`.

```go
package app

import "github.com/tinywasm/fmt"

// ormc:formonly
type stateResponse struct {
	JSONRPC string      `json:"jsonrpc"`
	ID      fmt.RawJSON `json:"id"`
	Result  fmt.RawJSON `json:"result"`
}
```

- `ID fmt.RawJSON` — el `id` extraído del request ya es JSON crudo (ej: `"1"` con comillas).
  Con `FieldRaw` se emite verbatim, sin doble encoding.
- `Result fmt.RawJSON` — `stateJSON` es el array JSON crudo de handlers. Se emite verbatim.

## Paso 2 — Instalar `ormc` y generar `app/model_orm.go`

```bash
go install github.com/tinywasm/orm/cmd/ormc@latest

cd tinywasm/app
ormc
```

`ormc` genera `app/model_orm.go` con `Schema()` y `Pointers()` para `stateResponse`.
El campo `Result` tendrá `Type: fmt.FieldRaw` porque su tipo es `fmt.RawJSON`.

Resultado esperado en `model_orm.go`:
```go
// DO NOT EDIT. generated by github.com/tinywasm/orm

package app

import (
    "github.com/tinywasm/fmt"
    "github.com/tinywasm/form/input"
)

var _schemastateResponse = []fmt.Field{
    {Name: "jsonrpc", Type: fmt.FieldText, Widget: input.Text()},
    {Name: "id",      Type: fmt.FieldRaw},
    {Name: "result",  Type: fmt.FieldRaw},
}

func (m *stateResponse) Schema() []fmt.Field { return _schemastateResponse }

func (m *stateResponse) Pointers() []any {
    return []any{&m.JSONRPC, &m.ID, &m.Result}
}
```

## Paso 3 — Actualizar `daemon.go`: reemplazar `fmt.Sprintf`

```go
// daemon.go — antes (~línea 144–155)
var stateJSON []byte
if projectTui != nil {
    stateJSON = projectTui.GetHandlerStates()
} else {
    stateJSON = ui.GetHandlerStates()
}

resp := fmt.Sprintf(`{"jsonrpc":"2.0","id":%s,"result":%q}`,
    id, string(stateJSON))
w.Header().Set("Content-Type", "application/json")
w.Write([]byte(resp))

// daemon.go — después
var stateJSON []byte
if projectTui != nil {
    stateJSON = projectTui.GetHandlerStates()
} else {
    stateJSON = ui.GetHandlerStates()
}

sr := stateResponse{
    JSONRPC: "2.0",
    ID:      fmt.RawJSON(id),
    Result:  fmt.RawJSON(string(stateJSON)),
}
var respBytes []byte
twjson.Encode(&sr, &respBytes)
w.Header().Set("Content-Type", "application/json")
w.Write(respBytes)
```

`twjson` ya está importado en `daemon.go` como `twjson "github.com/tinywasm/json"`.

---

## Protocolo antes vs después

| | Antes | Después |
|---|---|---|
| Respuesta daemon | `{"result":"[{\"tab_title\":\"BUILD\"...}]"}` | `{"result":[{"tab_title":"BUILD"...}]}` |

---

## Pasos de ejecución (orden estricto)

```bash
go install github.com/tinywasm/orm/cmd/ormc@latest
```

1. Crear `app/model.go` con `stateResponse` (ver Paso 1)
2. `cd tinywasm/app && ormc` → genera `app/model_orm.go`
3. Editar `app/daemon.go`: reemplazar bloque `fmt.Sprintf` (ver Paso 3)
4. `go build ./...`
5. `go test ./...`

## Cobertura de tests requerida

Agregar en `app/test/`:

```go
func TestStateResponse_EncodesCorrectly(t *testing.T) {
    stateJSON := []byte(`[{"tab_title":"BUILD","handler_name":"WasmClient","handler_type":1}]`)
    id := `"1"`

    sr := stateResponse{
        JSONRPC: "2.0",
        ID:      fmt.RawJSON(id),
        Result:  fmt.RawJSON(string(stateJSON)),
    }
    var respBytes []byte
    twjson.Encode(&sr, &respBytes)

    // Verificar JSON válido con result como array (no string)
    var envelope map[string]any
    if err := stdjson.Unmarshal(respBytes, &envelope); err != nil {
        t.Fatalf("invalid JSON: %v\noutput: %s", err, respBytes)
    }

    arr, ok := envelope["result"].([]any)
    if !ok || len(arr) == 0 {
        t.Fatalf("result debe ser array no vacío, got: %T %v", envelope["result"], envelope["result"])
    }

    entry := arr[0].(map[string]any)
    if entry["tab_title"] != "BUILD" {
        t.Errorf("tab_title: got %v, want BUILD", entry["tab_title"])
    }
}
```

> El test usa `encoding/json` solo para **validación** del output. El código de producción
> usa exclusivamente `tinywasm/json`.
